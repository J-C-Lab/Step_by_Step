## 功能三：面板拖动后可视化内容自适应（高优先级）

### 问题

使用 `Allotment` 拖动面板时，React Flow 画布容器变大/变小，但节点仍保持原始位置和缩放，导致节点偏小或溢出。

### 方案

在 `GraphCanvasInner` 内对画布容器 div 挂载 `ResizeObserver`，尺寸变化时防抖 150ms 后自动调用 `fitView`。

### 修改文件

**`src/visualizer/VisualizerView.jsx`**

- 新增 `containerRef`、`roTimerRef` 两个 ref
- 新增 `useEffect`：挂载 `ResizeObserver` 监听容器尺寸变化，防抖 150ms 触发 `fitViewRef.current({ padding: 0.2, duration: 250 })`（利用已有的稳定 ref，不引入新依赖）
- 为外层容器 div 绑定 `ref={containerRef}`

```javascript
useEffect(() => {
  const el = containerRef.current
  if (!el) return
  const ro = new ResizeObserver(() => {
    clearTimeout(roTimerRef.current)
    roTimerRef.current = setTimeout(() => {
      if (renderNodesRef.current?.length > 0) {
        fitViewRef.current?.({ padding: 0.2, duration: 250 })
      }
    }, 150)
  })
  ro.observe(el)
  return () => { ro.disconnect(); clearTimeout(roTimerRef.current) }
}, [])
```

## 功能二：Step Over / Step Out 实现（中优先级）

### 问题

工具栏中 Over / Out 两个按钮仅显示 toast "coming soon"，未实际接入沙盒。

### 方案

在 `InterpreterController.js` 新增三个函数，利用 stateStack 中 `FunctionExpression` / `FunctionDeclaration` 节点数量衡量当前调用栈深度。

### 修改文件

**`src/core/InterpreterController.js`**

新增：

| 函数 | 语义 |
|------|------|
| `getCallDepth(interp)` | 计算当前调用栈深度（stateStack 中函数节点数） |
| `stepOver()` | 执行直到调用深度 ≤ 进入时深度 且 行号变化 |
| `stepOut()` | 执行直到调用深度 < 进入时深度（顶层时退化为普通 step）|

两个函数均遵循与 `step()` 相同的快照捕获和状态更新流程，限制最大内部步数防止死循环。

**`src/components/Toolbar.jsx`**

- Over / Out 按钮 `onClick` 绑定 `Controller.stepOver()` / `Controller.stepOut()`（移除旧的 toast 回调）
- 更新 tooltip 说明：
  - Over：`单步跨过（执行完当前行的函数调用，停在下一行）`
  - Out：`跳出函数（执行完当前函数的剩余部分，返回调用处）`
- 同时移除孤立的 `toast` 辅助函数（已无使用处）

## 修复追加（同日用户反馈）

### 问题一：历史记录弹窗不可见（被代码区挡住）

**根因**：`HistoryPanel` 虽使用 `position: fixed`，但 `Toolbar` 的上级容器同时带有 `overflow-hidden` + `backdrop-filter: blur`（Tailwind `backdrop-blur-xl`）。CSS 规范中，`backdrop-filter` 非 `none` 时会为该元素建立新的 containing block，导致 `fixed` 子孙实际定位基准变为该祖先而非视口，且 `overflow-hidden` 将其裁剪。

**修复**：在 `HistoryPanel.jsx` 中引入 `react-dom` 的 `createPortal`，将弹窗渲染到 `document.body` 直属，彻底跳出任何可能干扰的祖先 CSS 上下文。

```javascript
import { createPortal } from 'react-dom'
// ...
return createPortal(panel, document.body)
```

**修改文件**：`src/components/HistoryPanel.jsx`

### 问题二：拖动布局后可视化视图重置为全局视图

**根因**：上一版 `ResizeObserver` 回调直接调用 `fitView()`，该操作会将视口重置为"展示全部节点"的状态，破坏用户已聚焦在某个变量上的缩放/平移状态。

**修复**：改为**等比例缩放当前视口**，保持用户的聚焦区域不变：

1. 额外引入 `getViewport`、`setViewport` 并通过 `useLayoutEffect` 保持稳定 ref
2. ResizeObserver 记录上一次容器尺寸 `prevSizeRef`
3. 每次尺寸变化时计算 `scale = sqrt(newW/oldW * newH/oldH)`（几何平均，均衡处理宽高变化）
4. 以屏幕中心点为锚点，按 `scale` 缩放当前 zoom 并重新计算 pan，调用 `setViewport`

关键效果：
- 面板变大 → 节点视觉尺寸变大，但焦点区域不变
- 面板变小 → 节点视觉尺寸缩小，但焦点区域不变
- **不再调用 `fitView`，永远不会重置到全局视图**

```javascript
const scale = Math.sqrt((newW / oldW) * (newH / oldH))
const vp = getViewportRef.current?.()
if (vp) {
  const { x, y, zoom } = vp
  const newZoom = zoom * scale
  const canvasCX = (oldW / 2 - x) / zoom
  const canvasCY = (oldH / 2 - y) / zoom
  setViewportRef.current?.({
    x: newW / 2 - canvasCX * newZoom,
    y: newH / 2 - canvasCY * newZoom,
    zoom: newZoom,
  })
}
prevSizeRef.current = { w: newW, h: newH }
```

**修改文件**：`src/visualizer/VisualizerView.jsx`
