## 主要内容
排查并修复二叉树示例在右侧可视化中被当作数组显示、以及输入树算法时需要手工建树导致无法继续调试的问题。

---

## 用户反馈场景

用户输入示例：

```javascript
function TreeNode(val, left, right) {
  this.val = (val === undefined ? 0 : val)
  this.left = (left === undefined ? null : left)
  this.right = (right === undefined ? null : right)
}
var flatten = function (root) {
  let prev = null;
  const traverse = (node)=>{
    if(!node) return;
    traverse(node.right);
    traverse(node.left);
    node.right = prev;
    node.left = node;
  }
  traverse(root);
};
let root = [1, 2, 5, 3, 4, null, 6];
```

现象：
- `root` 是层序数组时会被当成普通数组结构渲染，不显示 `left/right` 树边。
- 用户未手工建树时，算法很难进入预期的树处理路径。
- 该片段还存在未调用 `flatten(root)` 的情况，执行流不会进入算法主体。

---

## 根因分析

1. 可视化层仅把「对象形态的 `{val,left,right}`」识别为 tree，`[1,2,5,...]` 仍按 array 渲染。
2. 运行前没有自动做树输入转换；只有用户手动“适配可视化”时才可能补齐调用/输入。
3. 工具栏 Run 直接把原始代码送入解释器，未走预处理流程。

---

## 修复方案

### 1) 运行时自动预处理（无需每次手动点适配）

**文件：`src/core/InterpreterController.js`**

- 在 `init(code)` 中引入并执行 `prepareCodeForVisualization(code)`；
- 再将处理后的代码交给 Babel + js-interpreter。

效果：用户直接 Run/Step 时也会自动应用“补输入、补调用、树构建”等规则。

### 2) 自动把层序数组转为 TreeNode 链接结构

**文件：`src/utils/codePrep.js`**

- 新增 `autoBuildTreeInputs(code)`：
  - 检测代码中是否存在 `TreeNode`；
  - 对形如 `var root = [1,2,3,null,4];` 的典型层序输入自动改写为  
    `var root = __buildTreeFromLevelOrder([...]);`
  - 自动注入 `__buildTreeFromLevelOrder` 辅助函数，用 `new TreeNode` 生成节点并连 `left/right`。
- 将该步骤接入 `prepareCodeForVisualization` 主流程。

### 3) 可视化兜底识别：数组也可按树展示

**文件：`src/visualizer/VisualizerAdapter.js`**

- 在结构检测里新增层序树数组判定（变量名 + 数组形态启发式）；
- 命中后把数组转换成含 `__id__` 的树对象，再按 `type: 'tree'` 输出。

效果：即使代码未被改写，右侧画布也能按真实树结构绘制 `left/right`，不再只显示数组卡片。

---

- 工具栏直接运行也会触发预处理，不再强依赖手动点“适配可视化”。
