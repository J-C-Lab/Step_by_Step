## 主要内容
修复「每日温度」类算法片段在「适配可视化」后只能单步 3 次就提前结束的问题。

---

## 问题现象

用户输入 LeetCode 函数体片段（引用 `temperatures` 但未声明输入数据）：

```javascript
let answer = new Array(temperatures.length).fill(0);
let max = temperatures[0], min = temperatures[0];
for (let i = 0; i < temperatures.length - 2; i++) {
  for (let j = i + 1; j < temperatures.length - 1; j++) {
    if (temperatures[i] < temperatures[j]) {
      answer[i] = j - i;
      break;
    }
  }
}
return answer;
```

适配后代码在初始化循环访问 `temperatures.length` 时抛出 `ReferenceError: temperatures is not defined`，`InterpreterController.stepToNextLine()` 捕获异常并将状态设为 `finished`，表现为只能 Step 约 3 次（初始态 + `var answer = []` + 进入 for 循环）后无法继续。

---

## 根因

`prepareCodeForVisualization` 已有 `appendMissingFunctionInvocation` 为**函数包装**自动补全示例参数，但对**裸算法片段**（无 `function` 外壳、直接引用外部输入变量）未做处理。用户从 LeetCode 复制函数体时，`temperatures` 等形参不会出现在全局作用域，沙盒执行即失败。

---

## 修复方案

**文件：`src/utils/codePrep.js`**

1. 新增 `injectMissingTopLevelInputs(code)`：
   - 扫描已声明变量（`var`、逗号声明、`for (var ...)`、函数形参）；
   - 扫描代码中引用的标识符（剔除注释/字符串）；
   - 对未声明且属于已知算法输入名的变量（如 `temperatures`、`nums`、`grid` 等），在代码顶部注入 `var name = <示例值>;`。
2. 在 `prepareCodeForVisualization` 流程中，于 `appendMissingFunctionInvocation` 之前调用上述逻辑。
3. `getDefaultArgValue` 新增 `temperatures` 默认值：`[73, 74, 75, 71, 69, 72, 76, 73]`（LeetCode 739 经典样例）。

适配后示例输出：

```javascript
var temperatures = [73, 74, 75, 71, 69, 72, 76, 73];

var answer = [];
for (var __init_answer = 0; __init_answer < temperatures.length; __init_answer++) {
  answer[__init_answer] = 0;
}
// ... 后续算法逻辑 ...
var result = answer;
```
