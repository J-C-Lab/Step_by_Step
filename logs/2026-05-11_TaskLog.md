# 2026-05-11 工作日志

## 问题：minPathSum 适配后仍无法沙盒推演

### 根因分析

用户提交的 `minPathSum` 示例，点击"适配可视化"后代码结构正确，但沙盒依然无法执行推演。原因有三条叠加：

1. **函数参数没有全局定义**  
   `minPathSum(grid)` 中的 `grid` 只是函数形参，没有任何 `var grid = ...` 的全局赋值。`appendMissingFunctionInvocation` 发现参数不在全局域就直接 `return`，导致函数永远不被调用，算法主体从不执行，`dp/m/n` 等变量全部为空。

2. **函数只定义没调用**  
   直接后果：沙盒里只有一条 `var minPathSum = function(...){...};`，步进永远结束于第 1 步。

3. **`Infinity` 在 js-interpreter 沙盒内不稳定**  
   `new Array(n).fill(Infinity)` 展开后每个单元格都被赋值 `Infinity`，部分沙盒路径下对 `Infinity` 字面量的处理不一致，引起 DP 比较逻辑错误。

### 修改内容

**文件：`src/utils/codePrep.js`**

#### 1. `appendMissingFunctionInvocation` 重写

- 返回类型从 `string` 改为 `{ code, message }` 对象。
- 遇到参数未在全局定义时，**不再放弃**，改为调用 `getDefaultArgValue(paramName)` 自动生成示例数据并在末尾插入 `var param = <sample>;`。
- 最终追加 `var result = fn(params);` 调用。
- toast 提示语包含所有自动补充的参数名，提示用户按需修改。

#### 2. 新增 `getDefaultArgValue(name)` 辅助函数

按参数名（忽略大小写）推断合理默认值：

| 参数名 | 默认值 |
|---|---|
| `grid` / `matrix` | `[[1,3,1],[1,5,1],[4,2,1]]` |
| `nums` / `arr` / `array` / `values` / `prices` | `[1,5,11,5]` |
| `weights` / `wt` | `[1,2,3,5]` |
| `s` / `str` / `text` | `"abcde"` |
| `t` | `"ace"` |
| `n` / `capacity` / `target` | `5` |
| `m` | `3` |
| `k` | `2` |
| 其他 | `5` |

#### 3. 新增 `replaceInfinity(code)` 规则

在 `lightFormat` 之前执行，将代码中独立出现的 `Infinity` 字面量替换为 `1000000000`，避免 js-interpreter 对 IEEE Infinity 的不一致处理。  
不影响 `-Infinity`、`Number.POSITIVE_INFINITY` 等其他形式。

#### 4. `prepareCodeForVisualization` 调用链更新

- `appendMissingFunctionInvocation` 调用改为接收 `{ code, message }` 返回值。
- 在所有规则之后、`lightFormat` 之前插入 `replaceInfinity` 步骤。

### 适配后 minPathSum 示例代码预期输出

```javascript
var minPathSum = function(grid) {
    var m = grid.length;
    var n = grid[0].length;
    var dp = [];
    for (var __init_dp_row = 0; __init_dp_row < m; __init_dp_row++) {
      dp[__init_dp_row] = [];
      for (var __init_dp_col = 0; __init_dp_col < n; __init_dp_col++) {
        dp[__init_dp_row][__init_dp_col] = 1000000000;
      }
    }
    var preSum = 0;
    for (var i = 0; i < m; i++) {
        dp[i][0] = preSum + grid[i][0];
        preSum += dp[i][0];
    }
    preSum = grid[0][0];
    for (var i = 1; i < n; i++) {
        dp[0][i] = preSum + grid[0][i];
        preSum += dp[0][i];
    }
    for (var i = 1; i < m; i++) {
        for (var j = 1; j < n; j++) {
            dp[i][j] = Math.min(grid[i][j] + dp[i - 1][j], grid[i][j] + dp[i][j - 1]);
        }
    }
    return dp[m - 1][n - 1];
};

var grid = [[1, 3, 1], [1, 5, 1], [4, 2, 1]];
var result = minPathSum(grid);
```

## 功能：∞ / -∞ 显示优化

### 问题

`replaceInfinity` 将代码中的 `Infinity` 替换为 `1000000000`，初始化 DP 数组时每个格子都显示 `1000000000`，字符串过长导致矩阵格子溢出重叠。

### 修改内容

**文件：`src/visualizer/VisualizerView.jsx`**

在 `fmtVal` 函数中新增数值检测逻辑：
- 值 `=== Infinity` 或 `>= 1_000_000_000` → 显示 `∞`
- 值 `=== -Infinity` 或 `<= -1_000_000_000` → 显示 `-∞`

同时定义 `INF_SENTINEL = 1_000_000_000` 和 `NINF_SENTINEL = -1_000_000_000` 常量，使意图清晰。

由于所有渲染路径（数组格子、矩阵 `MatrixCell`、对象属性、原始值）都经过 `fmtVal`，此修改一次性覆盖所有场景。

## Bug 修复：左侧点击变量→右侧视图不跳转

### 根因

`VisualizerView.jsx` 的选择跳转 `useEffect` 依赖数组包含了 `renderNodes` 和 `fitBounds`，两个不稳定来源叠加导致跳转失效：

1. **`fitBounds` 引用不稳定**：`useReactFlow()` 每次渲染可能返回新的 `fitBounds` 引用，effect 被高频重新触发。
2. **`requestAnimationFrame` 没有 cleanup**：每次 effect 重新触发时，上一次尚未执行的帧回调不被撤销，旧回调在 React Flow 内部状态变化后执行，`fitBounds` 调用无效。
3. **`renderNodes` 每步都变**：随着用户点击"下一步"，effect 重复竞争触发，实际跳转被覆盖丢失。

### 修复方案（`src/visualizer/VisualizerView.jsx`）

使用 **`useRef` + `useLayoutEffect`** 模式：将 `fitBounds`、`fitView`、`renderNodes` 固定到 ref，`useLayoutEffect` 在每次渲染后同步更新，确保 effect 访问最新值。

选择 effect 只依赖 `selectedVariable` 和 `ready`，彻底脱离 `renderNodes` / `fitBounds` 的引用变化干扰，并为 `requestAnimationFrame` 添加 cleanup。

**关键改进：**
- effect 只在 `selectedVariable` 或 `ready` 变化时触发
- `requestAnimationFrame` 有 cleanup，防止旧回调失效
- `useLayoutEffect` 确保 refs 总是最新值

### 验证

- `npm run build`：通过，无新增错误。
- `src/visualizer/VisualizerView.jsx` lint：无问题。

## Bug 修复：`minimumTotal` 三角形 DP 例子无法沙盒推演

### 根因

```javascript
var dp = Array.from({ length: m }, (_, i) => new Array(i + 1).fill(1000000000));
```

`expandArrayFromFill` 的正则只匹配 `() =>` 无参箭头函数（所有行等长）。  
这里是 `(_, i) =>` 带行索引的箭头函数，列数依赖 `i`（锯齿/三角形数组）。  
规则匹配失败 → `Array.from` 原样留在代码中 → `js-interpreter` 不支持 `Array.from` 的映射函数参数 → 运行崩溃。

### 修改内容

**文件：`src/utils/codePrep.js`，函数 `expandArrayFromFill`**

在原有 Pattern A（固定列数）的基础上，新增 Pattern B（行索引相关的锯齿形数组）：

| 模式 | 原始形式 | 展开结果 |
|------|---------|---------|
| Pattern A | `Array.from({length:N}, () => new Array(M).fill(V))` | 等长二维数组初始化双重 for 循环 |
| Pattern B | `Array.from({length:N}, (_, i) => new Array(EXPR_i).fill(V))` | 锯齿形二维数组初始化，列数为 `EXPR_i`（`i` 替换为实际行循环变量名） |

`minimumTotal` 例子中 `Array.from({length:m}, (_, i) => new Array(i+1).fill(1000000000))` 经 Pattern B 展开为：

```javascript
var dp = [];
for (var __init_dp_row = 0; __init_dp_row < m; __init_dp_row++) {
  dp[__init_dp_row] = [];
  for (var __init_dp_col = 0; __init_dp_col < __init_dp_row + 1; __init_dp_col++) {
    dp[__init_dp_row][__init_dp_col] = 1000000000;
  }
}
```

### 验证

- `npm run build`：通过，无新增错误。
- `src/utils/codePrep.js` lint：无问题。
