## 主要内容
为二叉树可视化新增「当前计算节点高亮」，单步执行时可清晰看到算法正在访问哪个节点。

---

## 问题

二叉树已能正确绘制，但 Step 过程中无法直观判断算法当前正在处理哪个节点，不利于理解 DFS/BFS/递归遍历等算法的执行路径。

---

## 方案

从 timeline 快照的变量中解析「当前指针」对应的节点 `__id__`，并在 React Flow 画布上对该节点施加高亮样式。

### 指针识别规则（`resolveActivePointerIds`）

1. 扫描当前步 `variables` 中所有指向树节点或链表节点的对象（需有 `__id__`）。
2. 按常见遍历变量名优先级选取：`node` > `cur` > `current` > `p` > `ptr` > … > `head` > `root`。
3. 若无匹配变量名，则对比前后两步快照，高亮指针身份发生变化的节点。

### 视觉表现（`GlassNode` 树节点）

- 绿色粗边框 + 外发光环
- 浅绿背景
- 右上角 ▶ 标记
- 优先级：`isCurrent` > `isSelected` > `isActive`

## 使用说明

编写树算法时，用 `node`/`cur`/`root` 等变量保存当前访问节点即可自动高亮。例如：

```javascript
function dfs(root) {
  if (!root) return;
  // 此时 root 对应节点会高亮
  dfs(root.left);
  dfs(root.right);
}
```

或迭代写法：

```javascript
var node = root;
while (node) {
  // node 所指节点会逐步高亮
  node = node.left;
}
```

## 追加修复（同日反馈：树节点不相连）

### 时间
2026-07-09 12:11:00

### 问题

画布上出现两个孤立圆形节点（如 15 和 7），彼此没有 `left/right` 连线，不符合从 `root` 层层向下渲染的预期。

### 根因

同一时刻多个变量被同时识别为独立 `tree` 结构并分别渲染，例如：

- `root`：解释器中的真实 `TreeNode`（`obj_*` id）
- `nodes`：建树过程中的节点池数组（合成 `nodepool_*` id）
- `node` / `cur`：遍历指针（也被当成独立树）

这些结构的 `__id__` 互不重叠，`coveredPointerIds` 去重失效，导致同一逻辑节点被画两次且边无法连接。

### 修复（`filterRenderStructures`）

在 `buildVisualizerState` 输出前统一筛选：

1. 存在真实 `TreeNode` 树时，丢弃 `arrtree_*` / `nodepool_*` 合成树；
2. 存在 `root` / `tree` 等 canonical 变量时，忽略 `node` / `cur` 等 walker 指针的独立渲染（仍用于高亮）；
3. 多棵树并存时只保留节点数最多的一棵；
4. 已有 canonical 树时，隐藏冗余的层序数组卡片。

### 验证

- 模拟 `root + nodes + node` 同时存在：仅保留 `root` 一棵树，`npm run build` 通过。
