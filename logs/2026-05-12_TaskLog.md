# 2026-05-12 工作日志

## 问题：同类 DP 算法（`uniquePaths` 等）适配后仍只跑 2-3 步就结束

### 现象

用户提交 `uniquePaths` 示例：

```javascript
var uniquePaths = function(m, n) {
    let dp = Array.from({length: m}, () => new Array(n).fill(0)); // 到当前格子所拥有的路径数
    for(let i = 0; i < m; i++){
        dp[i][0] = 1;
        dp[0][i] = 1;
    }
    for(let i = 1; i < m; i++){
        for(let j = 1; j < n; j++){
            dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
        }
    }
    return dp[m - 1][n - 1];
};
```

点击"适配可视化"后，沙盒推演只走 2-3 步即停止。

---

### 排查过程

逐层追溯执行链路：

1. **`InterpreterController.js`**：`stepToNextLine()` 的 `catch` 块一旦捕获运行时错误，立即调用 `setStatus('finished')` 并返回 `false`，导致沙盒停止。

2. **`codeTransformer.js`**：Babel 负责把 ES6+ 转为 ES5，但 `Array.from` 的第二个参数（mapper 函数）在 `js-interpreter` 的 ES5 运行环境中依然不被支持，哪怕 Babel 把箭头函数编译成普通函数。

3. **`codePrep.js` 的 `expandArrayFromFill`**：本应把 `Array.from(...)` 展开为显式 for 循环，但匹配失败。

4. **根因定位**：用户代码行尾有中文注释 `// 到当前格子所拥有的路径数`。`expandArrayFromFill`（及同类规则）的正则以 `\s*;?\s*$` 严格匹配行尾，注释打破了该匹配，展开规则静默跳过，`Array.from` 原样进入沙盒，抛运行时错误，沙盒 2-3 步即终止。

---

### 根因总结

**`codePrep.js` 四个行扫描正则对行内注释不兼容**，是所有同类 DP 算法"适配后仍只跑 2-3 步"的唯一根因：

| 正则结尾（修复前） | 问题 |
|---|---|
| `\s*;?\s*$` | 行内 `// comment` 使整行匹配失败 |
| `\{\s*$` | 同上 |

---

### 修改内容

**文件：`src/utils/codePrep.js`**

在以下四处行扫描正则末尾，统一追加 `(?:\/\/[^\r\n]*)?` 非捕获组，使行内注释被静默忽略：

| 函数 | 修改前末尾 | 修改后末尾 |
|---|---|---|
| `expandForOfLoops` | `\{\s*$` | `\{\s*(?:\/\/[^\r\n]*)?$` |
| `expandArrayFromFill` Pattern A | `\s*;?\s*$` | `\s*;?\s*(?:\/\/[^\r\n]*)?$` |
| `expandArrayFromFill` Pattern B | `\s*;?\s*$` | `\s*;?\s*(?:\/\/[^\r\n]*)?$` |
| `expandArrayFill` | `\s*;?\s*$` | `\s*;?\s*(?:\/\/[^\r\n]*)?$` |

修复后，不论行尾是否有注释，四个展开规则均可正常匹配并展开，`Array.from` 被替换为沙盒兼容的显式 for 循环。
