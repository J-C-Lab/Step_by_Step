## 追加记录（16:52 DP 示例只能执行两步）

### 用户反馈

适配后的 DP 示例虽然能进入沙盒，但只执行两步就结束，无法继续观察完整算法过程。

### 根因判断

问题出在：

```javascript
var dp = new Array(n).fill(0);
```

这类现代数组方法在当前 `js-interpreter` 沙盒环境里不够稳定，可能导致解释器快速结束或中断。语法适配成功并不等于运行时 API 一定可用。

### 修复内容

修改文件：`src/utils/codePrep.js`

新增规则：将 `new Array(n).fill(value)` 展开为 ES5 显式初始化循环。

例如：

```javascript
var dp = new Array(n).fill(0);
```

会被适配为：

```javascript
var dp = [];
for (var __init_dp = 0; __init_dp < n; __init_dp++) {
  dp[__init_dp] = 0;
}
```

这样沙盒会逐步执行初始化过程，`dp` 的变化也能被 Watch 和右侧可视化捕获。

### 修复后的示例输出

```javascript
var n = 5;
var dp = [];
for (var __init_dp = 0; __init_dp < n; __init_dp++) {
  dp[__init_dp] = 0;
}
dp[0] = 1;
dp[1] = 1;
for (var i = 2; i < n; i++) {
  dp[i] = dp[i - 1] + dp[i - 2];
}
var result = dp[n - 1];
```

### 验证

- 使用 DP 示例直接 smoke test：通过，输出已展开初始化循环。

## 追加记录（17:14 适配器误改复合赋值与复杂 DP 示例）

### 用户反馈

1. `适配可视化` 会把原本正确的：

```javascript
sum += num;
```

错误改成：

```javascript
sum + = num;
```

2. 分割等和子集示例仍无法正常运行可视化，代码包含：
   - `for...of`
   - `Array.from({length: n}, () => new Array(target + 1).fill(false))`
   - 函数只定义未调用
   - `var nums = nums = [...]` 重复赋值

### 根因

1. 轻量格式化中的赋值正则只排除了 `<=`、`>=`、`!==` 等情况，没有排除 `+=`、`-=`、`*=` 等复合赋值，导致 `+=` 被拆坏。
2. 当前沙盒更适合 ES5 风格代码，`for...of`、`Array.from`、箭头函数和 `.fill()` 的组合不适合直接交给执行沙盒。
3. 只定义 `canPartition` 函数但没有调用时，算法主体不会执行，局部变量 `sum/target/dp` 也不会进入 Watch 和可视化。

### 修复内容

修改文件：`src/utils/codePrep.js`

1. 修复复合赋值格式化
   - 避免将 `+=` 改成 `+ =`。
   - 对已经被错误适配过的 `+ =`、`- =`、`* =`、`/ =`、`% =` 做恢复。

2. 新增重复 self assignment 修复

```javascript
var nums = nums = [1, 5, 11, 5];
```

会被修正为：

```javascript
var nums = [1, 5, 11, 5];
```

3. 新增 `for...of` 转普通索引循环

```javascript
for (var num of nums) {
  sum += num;
}
```

会被改为：

```javascript
for (var __idx_nums = 0; __idx_nums < nums.length; __idx_nums++) {
  var num = nums[__idx_nums];
  sum += num;
}
```

4. 新增二维 `Array.from(...fill...)` 展开

```javascript
var dp = Array.from({length: n}, () => new Array(target + 1).fill(false));
```

会被改为显式二维初始化循环：

```javascript
var dp = [];
for (var __init_dp_row = 0; __init_dp_row < n; __init_dp_row++) {
  dp[__init_dp_row] = [];
  for (var __init_dp_col = 0; __init_dp_col < target + 1; __init_dp_col++) {
    dp[__init_dp_row][__init_dp_col] = false;
  }
}
```

5. 新增函数未调用检测
   - 识别 `var canPartition = function(nums) { ... };`
   - 如果存在全局 `nums` 且没有调用该函数，则自动追加：

```javascript
var result = canPartition(nums);
```

### 验证

- 使用 `sum += num` 示例 smoke test：不再被改坏。
- 使用用户提供的 `canPartition` 示例 smoke test：可生成 ES5 风格、显式初始化、自动调用的代码。
- `ReadLints` 检查 `src/utils/codePrep.js`：无新增问题。
- `npm run build`：通过。
